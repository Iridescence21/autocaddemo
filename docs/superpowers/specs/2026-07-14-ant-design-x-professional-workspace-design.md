# Ant Design X 专业图纸分析工作台设计

## 目标

将当前以基础布局和大量内联样式拼装的页面，重构为明亮、专业、响应式的三段式电气图纸分析工作台。界面优先使用 Ant Design X 2.8 已提供的组件、默认样式、语义插槽和主题令牌，仅为应用级布局补充少量 CSS Module。

工作台必须同时支持自然语言对话、CAD 分析过程、图纸文件管理、结构化识别结果、人工复核和 BOM 导出，避免把所有信息压缩成一条 Markdown 消息。

## 非目标

- 本次不修改 CAD 解析、视觉模型、物理设备分组或 BOM 生成算法。
- 本次不建立跨会话文件资料库。
- 不为凑组件数量强行加入没有业务内容的代码高亮或 Mermaid 图表。
- 不引入另一套 UI 组件库与 Ant Design X 竞争。

## 信息架构

页面采用三段式布局：

1. 左侧会话栏，桌面宽度约 260 像素。
2. 中央对话工作区，自适应占据主要空间。
3. 右侧工程详情栏，桌面宽度约 360 像素，可折叠。

左侧使用 `Conversations` 管理当前用户的分析会话，包含新建、激活、重命名和删除操作。顶部以紧凑品牌区展示“电气图纸 AI”和工作台说明，底部保留工程师复核提示。

中央区域由会话头、消息流和固定输入区组成。空会话显示 `Welcome` 与 `Prompts`；已有消息使用 `Bubble.List`。输入区使用 `Sender` 的 header、footer、switch、prefix 和 suggestion 能力组成完整的智能体输入体验。

右侧详情栏包含五个页签：

- 图纸：总览预览、尺寸、格式和分析状态。
- 文件：当前分析会话的文件集合。
- 元件：符号实例列表和物理设备数量。
- 复核：待确认、未知和低置信度对象。
- BOM：采购分组与导出操作。

窄屏下左侧会话栏和右侧详情栏分别变为可打开的抽屉，中央对话保持完整宽度。不再设置 `min-width: 1000px`。

## Ant Design X 组件编排

| 组件 | 使用位置 | 业务语义 |
| --- | --- | --- |
| `XProvider` | 应用根节点 | 中文本地化、主题令牌和统一圆角/色彩 |
| `Conversations` | 左侧栏 | 会话创建、切换和菜单操作 |
| `Welcome` | 空状态和品牌引导 | 产品介绍、当前会话摘要 |
| `Prompts` | 空状态和上下文建议 | 上传、识别、BOM、复核、结果视图快捷入口 |
| `Bubble.List` | 中央消息流 | 用户、AI、系统和结果消息 |
| `Bubble.System` | 系统状态 | 上传成功、分析失败、覆盖受限等系统提示 |
| `Bubble.Divider` | 消息分段 | 分析轮次、日期或重新分析边界 |
| `Think` | AI 消息内部 | 可展开的模型分析思路与识别依据摘要 |
| `ThoughtChain` | CAD 任务消息内部 | 准备、转换、切图、识别、去重、生成清单等真实阶段 |
| `Attachments` | Sender header | 拖拽、粘贴和选择 DWG/DXF |
| `Sender` | 中央底部 | 自然语言和指令输入 |
| `Sender.Header` | 输入区展开面板 | 当前待上传附件与错误状态 |
| `Sender.Switch` | 输入区 footer | “完整识别”“仅复核”等可用分析模式；仅连接真实能力 |
| `Suggestion` | 输入区快捷指令 | `@分析`、`@筛选`、`@复核`、`@BOM`、`@导出` |
| `Actions` | 消息和结果卡片底部 | 复制、重试、定位、确认、编辑、移除和导出 |
| `FileCard` | 消息及文件区 | 原始图纸、预览和导出文件 |
| `Folder` | 文件区 | 按原始文件、分析产物、导出结果分组 |
| `Sources` | 元件详情 | 图层、块名、文字证据、瓦片和检测来源 |
| `XNotification` | 非阻塞反馈 | 保存、导出、重新分析等操作结果 |
| `XMarkdown` | 文本消息 | 摘要、说明和表格，不承担整个工作台布局 |

`Think` 与 `ThoughtChain` 有明确分工：前者描述 AI 如何判断，后者描述系统实际执行到哪一步。两者不可用同一份虚构文案代替。

## 当前会话文件区

文件区只展示当前分析会话的数据，不跨会话聚合。内容按三个 `Folder` 分组：

1. 原始图纸：用户上传的 DWG 或 DXF。
2. 分析产物：图纸总览、分析瓦片以及可公开给用户的转换结果。
3. 导出结果：元件分析 Excel 和 CSV。

每个文件用 `FileCard` 展示名称、类型、大小和状态。可用操作由数据决定：预览、下载、重新分析或移除待上传附件。未被后端持久化或没有安全下载接口的内部临时文件不得显示为可下载文件。

聊天中的文件卡片与右侧文件区共享同一展示模型。点击卡片会切换到文件页签并定位对应文件。

## 消息与结果设计

用户消息使用右侧 filled/round 气泡和用户头像。AI 消息使用左侧无边框或浅色气泡、产品头像、头部角色名、底部操作区。系统消息使用 `Bubble.System`，轮次边界使用 `Bubble.Divider`。

消息类型按现有数据映射：

- `text`：`Bubble` + `XMarkdown`。
- `file`：`Bubble` + `FileCard`。
- `analysis_progress`：`Bubble` + `ThoughtChain`。
- `drawing_summary`：AI 气泡，包含可展开 `Think`、摘要和覆盖警告。
- `component_results`：AI 结果气泡，展示关键指标和进入右侧元件详情的操作。
- `bom_results`：AI 结果气泡，展示分组与总数量并提供导出操作。
- `error`：`Bubble.System` 错误状态和重试操作。

元件详情、复核和 BOM 不再在消息流中展开超长 `Prompts` 或 Markdown 表格。消息只提供摘要，完整操作在右侧详情栏完成。

## 样式策略

- 保留 Ant Design X 官方组件 CSS 和交互状态，不复制官方示例的内部 DOM 样式。
- 通过 `XProvider` 主题令牌设置主色、背景、圆角、字体和阴影。
- 使用 CSS Module 负责页面网格、响应式断点、面板尺寸和少量品牌装饰。
- 使用组件的 `variant`、`shape`、`placement`、`header`、`footer`、`classNames` 和语义插槽进行定制。
- 删除主工作台中的大部分内联样式，避免固定高度、强制最小宽度和散落的颜色值。
- 视觉基调为白色与浅灰背景、蓝色强调、克制阴影和清晰层级，不使用霓虹边框。

## 状态与错误处理

- 发送中、取消、禁用和附件错误直接反映在 `Sender` 状态中。
- 分析进行中时 `ThoughtChain` 保持可见，完成后保留阶段历史但默认折叠。
- 覆盖受限、瓦片失败和未知元件必须持续可见，不能因界面美化被隐藏。
- 保存、导出、确认和编辑使用非阻塞通知；破坏性删除仍需确认。
- 空会话、无文件、无元件、无待复核项和无 BOM 分别提供明确空状态。

## 可访问性

- 所有图标按钮提供可读标签和 tooltip。
- 颜色不是状态的唯一表达方式，状态同时显示文字或图标。
- 桌面与窄屏均保持键盘可达，抽屉打开后焦点进入面板。
- 气泡、文件和结果操作保持合理触控尺寸。

## 代码边界

当前 550 行的 `drawing-workspace.tsx` 拆分为职责清晰的组件：

- `DrawingWorkspace`：数据加载和业务动作编排。
- `WorkspaceShell`：三段式响应式布局。
- `ConversationSidebar`：会话列表。
- `WorkspaceWelcome`：空状态。
- `DrawingMessageList`：消息类型到 Bubble 的映射。
- `AnalysisComposer`：Sender、附件和 Suggestion。
- `WorkspaceInspector`：右侧页签。
- `SessionFilesPanel`：当前会话文件模型与展示。
- `ComponentInspector`、`ReviewInspector`、`BomInspector`：结构化结果操作。

组件只通过显式 props 接收数据和回调，网络请求继续由工作区容器协调，避免展示组件直接拼 API 地址。

## 测试

- 用纯函数测试消息类型到 Bubble 角色、状态和内容模型的映射。
- 测试当前会话文件分组，确保不混入其他会话文件。
- 测试空状态、分析中、失败、完成和覆盖受限的展示模型。
- 测试 Sender 附件、Suggestion 指令和右侧页签联动。
- 保留现有中文文案、符号/物理设备区分和 Excel 导出断言。
- 运行 Vitest、TypeScript、ESLint 和 Webpack 生产构建。
- 使用浏览器分别检查空会话、已上传、分析中、完成结果和窄屏布局。

## 验收标准

- 页面在桌面上呈现左会话、中央对话、右详情的专业工作台布局。
- 当前会话文件区按原始图纸、分析产物和导出结果分组。
- 与业务相关的 Ant Design X 组件得到真实使用，而非仅出现在 import 中。
- AI 思考与 CAD 任务进度分别由 `Think` 和 `ThoughtChain` 表达。
- 消息气泡具有头像、角色、状态和操作，不再是裸文本堆叠。
- Sender 具备附件面板、快捷指令、真实可用开关和清晰发送状态。
- 结果摘要与右侧结构化详情联动，长列表不再阻塞聊天流。
- 窄屏可用，不再依赖 1000 像素最小宽度。
- 不削弱工程师复核、覆盖受限和未知元件警告。
