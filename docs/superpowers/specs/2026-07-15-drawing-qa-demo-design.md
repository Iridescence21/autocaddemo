# 天智图纸智能问询演示设计

## 目标

用户分别上传 `M-T1-01.dwg` 和 `M-T1-02.dwg` 并完成分析后，可以在现有聊天框中用自然语言查询图纸设备、型号、数量和跨图纸分布，生成可见 BOM，并执行基于原生 CAD 数据的基础一致性审查。

## 演示验收场景

系统必须能从真实 DWG 原生文字和表格中回答：

- “电流继电器有几种类型？”——当前图纸返回不重复型号及数量。
- “哪张图纸分布的电流继电器多？”——在当前用户已分析图纸中比较数量并列出每张图纸结果。
- “电流继电器在哪个图纸里面？”——列出包含该设备的图纸、数量、型号和代号。
- “生成这张图的 BOM 清单。”——将 CAD 表格识别结果写入现有 BOM，并在右侧 BOM 面板显示。
- “审查这张图纸。”——报告数量与代号数量不一致、型号缺失、数量缺失和同一代号信息冲突；不宣称已经完成电气拓扑或保护整定校核。

`M-T1-02.dwg` 的电流继电器验收基准为 2 种型号、7 只；`M-T1-01.dwg` 为 2 种型号、4 只。跨图纸问题必须得出 `M-T1-02.dwg` 更多，并说明两张图纸均包含电流继电器。

## 架构

采用“原生 CAD 结构证据优先，视觉识别补充”的混合方案：

1. 现有 DWG 渲染器将 DWG 转为 DXF 和图片分块。
2. 新增原生 BOM 表格提取器，从 `NormalizedDxfDrawing.texts` 的坐标、文字和表头关系中提取名称、型号规格、数量和设备代号。
3. 分析服务在调用视觉模型前持久化结构化快照。即使未配置大模型 API，问询和 BOM 仍可使用。
4. 新增确定性问询服务，解析设备名称、类型数、数量、分布、BOM 和审查意图；回答只引用持久化结构证据。
5. 新增会话问询 API，保存助手文本消息和证据数组。现有聊天界面的未匹配输入改为调用该 API。

## 数据模型

`Drawing` 新增 `structuralSnapshot Json?`，保存版本化快照：

- `schemaVersion`: 固定为 `1`。
- `counts`: 实体、文字、块、图层和结构标签数量。
- `tags`: 原生 CAD 设备代号证据。
- `bomRows`: 表格行，包含 `name`、`modelSpec`、`quantity`、`symbolTags`、坐标和句柄证据。
- `reviewIssues`: 基础一致性问题，包含严重度、说明和相关代号。

快照属于单张图纸；跨图纸查询只读取同一 `ownerScope` 的已持久化快照，不跨用户。

## 问询规则

- 设备名称匹配采用规范化的中文名称包含匹配；“电流继电器”只匹配 BOM 名称为电流继电器的行，不把所有 `KC` 或所有继电器混算。
- “几种类型/型号”按非空 `modelSpec` 去重，缺失型号单独提示，不臆造型号。
- 数量优先使用 BOM 数量；数量缺失时可退化为不重复设备代号数，并明确标注估算来源。
- “哪张/哪个图纸/在哪里/最多”触发同一用户范围的跨图纸聚合。
- 默认查询当前会话图纸；如果图纸尚未完成结构化分析，返回可操作提示，不调用大模型猜测。
- 回答附带 `evidence`，至少包含图纸名、BOM 行和设备代号。

## BOM 与审查

原生 BOM 行转换为现有 `BomItem`，分类由设备名称映射；未知类别保留中文描述并标记待复核。生成 BOM 不依赖视觉模型成功。

第一阶段审查只覆盖可由当前证据可靠验证的内容：

- BOM 数量与展开后的设备代号数量不一致。
- 型号规格为空。
- 数量为空或不是正整数。
- 同一图纸中同一代号对应多个名称或型号。

结构干涉、保护配合、回路连通性和公差合理性不在本阶段内，界面回答必须明确边界。

## 错误处理

- DWG 转换失败沿用现有中文错误码和状态。
- 结构快照成功、视觉模型失败时，分析状态可提示视觉能力受限，但结构问询仍可用。
- 无匹配设备时返回“未在已分析图纸中找到”，并列出参与查询的图纸数量。
- API 输入使用 Zod 校验，所有数据库读取受 `OWNER_SCOPE` 限制。

## 测试与验收

- 单元测试覆盖表格提取、代号展开、类型/数量/分布回答和基础审查。
- Repository/API 测试覆盖快照持久化、租户隔离和消息保存。
- UI 源码测试确保未匹配问题调用问询 API，而不是旧的演示占位回答。
- 真实烟测上传/分析 `M-T1-01.dwg` 与 `M-T1-02.dwg`，断言 4 对 7、2 种对 2 种和 `M-T1-02.dwg` 为最多。
- 完整执行测试、类型检查、Lint 和生产构建。
